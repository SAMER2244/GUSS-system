# 🚀 منظومة Antigravity V3.2 — مكتب التقييم والمتابعة

هذه المنظومة هي الأداة الرسمية لمكتب المتابعة و التقييم لدى **الاتحاد العام لطلبة سوريا (GUSS)**. تم بناء المنظومة باستخدام أحدث تقنيات الذكاء الاصطناعي لضمان أعلى درجات الدقة والاحترافية في العمل المؤسسي.

## 🎯 الهدف من المنظومة

- **أتمتة عمليات التدقيق والمتابعة** لتقارير الفروع بشكل كامل.
- **مطابقة التنفيذ الميداني مع الخطط الاستراتيجية (PDF)** بدقة متناهية لتقييم الأداء الفعلي.
- **دعم صناع القرار** بتقارير تحليلية معمقة يتم توليدها آلياً بأسلوب احترافي واستشاري.

## 🧠 محرك المعالجة المتوازي

جوهر النظام هو محرك معالجة يعمل بأربعة خيوط متوازية تستخدم نماذج Google Gemini لضمان سرعة المعالجة وجودة التحليل:

```text
تصدير البيانات (Google Sheets & Drive APIs)
      │
      ▼
[ محرك المعالجة المتوازي — 4 خيوط ]
  ├── خيط 1 [الملخص التنفيذي]   ➔ Gemini 3.5 Flash
  ├── خيط 2 [تحليل المهام]       ➔ Gemini 3.5 Flash
  ├── خيط 3 [التدقيق والمطابقة] ➔ Gemini 3.5 Flash
  └── خيط 4 [تحديات العمل]      ➔ Gemini 3.5 Flash
      │
      ▼
توليد تقرير Word & المزامنة السحابية (Drive OAuth2)
```

## 🛡️ نظام الصمود والطوارئ

1. **التراجع الثلاثي المستويات**: عند حدوث خطأ مؤقت (429 Resource Exhausted) على النموذج الافتراضي (`gemini-3.5-flash`)، يتم الانتقال تلقائياً إلى النموذج الاحتياطي (`gemini-3.1-flash-lite`). في حال استمرار الخطأ، يدخل النظام حلقة انتظار ذكية (30 ثانية بين كل محاولة) حتى استعادة الخدمة.
2. **تنظيف المدخلات**: يتم تنقية جميع النصوص المُرسَلة من أحرف التحكم والترميزات الفاسدة قبل إرسالها لنماذج الذكاء الاصطناعي.
3. **فترة تبريد إلزامية**: 10 ثوانٍ بين كل مكتب لتفادي تجاوز حدود استخدام واجهات البرمجة.

## ⚙️ الموجه التفاعلي

- **انتقاء المكاتب المستهدفة**: واجهة سطر أوامر تفاعلية تسمح باختيار مكاتب محددة لمعالجتها باستخدام معرّفات رقمية بسيطة (مثلاً: `1 3`) أو إدخال `all` لمعالجة كافة الداتا دفعة واحدة.
- **تخصيص شهر التقييم**: تجاوز آلي لتاريخ التقرير عبر إدخال رقم الشهر (1-12). يترجم النظام الإدخال تلقائياً لأشهر المشرق العربي (مثال: `4 ➔ نيسان`)، ويقوم بمزامنة الاسم ضمن أسماء ملفات (`.docx`)، صفحات الغلاف، والمجلدات المنشأة على Google Drive لضمان الامتثال التام للمعايير الإدارية.

## 🔧 دليل التثبيت وإعداد البيئة

تعتمد المنظومة على بيئة Miniconda لإدارة الاعتماديات البرمجية بشكل نقي ومعزول:

**1. استنساخ المستودع:**
```bash
git clone <repository_url>
cd <repository_name>
```

**2. تهيئة بيئة Miniconda الخاصة بـ الذكاء الاصطناعي وتفعيلها:**
```bash
conda env create -f environment.yml
conda activate ai_env
```

**3. إعداد بيانات الاعتماد والمفاتيح السرية (Credentials):**
تأكد من وضع ملفات الاعتماد الحساسة في المجلد الجذري للمشروع (تتجاهلها المنظومة تلقائياً ولن تُرفع للإنترنت بفضل إعدادات الأمان الصارمة المُجهزة مسبقاً في `.gitignore`):
- إنشاء ملف `.env` وإضافة مفتاح Gemini بداخله (`GEMINI_API_KEY=your_key_here`).
- توفير ملف `credentials.json` (ملف مصادقة Google Cloud Service Account).
- سيتم توليد ملف الوصول `token.json` آلياً عند تشغيل المنظومة والمصادقة على الحساب للمرة الأولى عبر المتصفح.

**4. تشغيل منصة العمل:**
```bash
python main.py
```
اتبع التعليمات التفاعلية الواضحة على واجهة أوامر التشغيل (Terminal) لاختيار المكاتب وتحديد شهر التقييم واسمح لمحرك الذكاء الاصطناعي بتولي المهمة! 🤖
