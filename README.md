# 🚀 منظومة Antigravity V3.1 — مكتب التقييم والمتابعة

هذه المنظومة هي الأداة الرسمية للتدقيق والمتابعة المعتمدة لدى **الاتحاد الوطني لطلبة سورية (NUSS)**. تم بناء المنظومة باستخدام أحدث تقنيات الذكاء الاصطناعي لضمان أعلى درجات الدقة والاحترافية في العمل المؤسسي.

## 🎯 الهدف من المنظومة (The 'Why')

- **أتمتة عمليات التدقيق والمتابعة** لتقارير الفروع بشكل كامل.
- **مطابقة التنفيذ الميداني مع الخطط الاستراتيجية (PDF)** بدقة متناهية لتقييم الأداء الفعلي.
- **دعم صناع القرار** بتقارير تحليلية معمقة يتم توليدها آلياً بأسلوب احترافي واستشاري.

## 🧠 محرك المعالجة المتوازي (The Parallel Orchestrator)

جوهر النظام هو محرك معالجة يعمل بأربعة خيوط متوازية لتوجيه المهام إلى النماذج اللغوية المتخصصة لضمان سرعة المعالجة وتقليل استهلاك الموارد:

```text
تصدير البيانات (Google Sheets & Drive APIs)
      │
      ▼
[ محرك المعالجة المتوازي — 4 خيوط ]
  ├── خيط 1 [الملخص التنفيذي] ➔ Groq (Llama-3.3-70b-Versatile)
  ├── خيط 2 [تحليل المهام]     ➔ Groq (Llama-3.3-70b-Versatile)
  ├── خيط 3 [التدقيق والمطابقة] ➔ Google (Gemini)
  └── خيط 4 [تحديات العمل]    ➔ Groq (Gemma)
      │
      ▼
توليد تقرير Word & المزامنة السحابية (Drive OAuth2)
```

## 🛡️ نظام الصمود والطوارئ (Failover & Resilience)

1. **التدوير الذكي للمفاتيح (Intelligent Key Rotation)**: تبدأ المنظومة بمجموعة من مفاتيح واجهة البرمجة (API Keys). عند استنفاذ الحصة المؤقتة (429 Resource Exhausted)، يقوم المحرك بالتدوير التلقائي نحو المفتاح التالي مع تطبيق تأخير زمني تصاعدي (Exponential Backoff) لحماية استمرارية العمل.
2. **بروتوكول الطوارئ الشامل (Global LLM Fallback)**: في حال تعطل أو استنفاذ جميع مسارات مزود Groq، يتم تحويل مسار البيانات فورياً وبشكل صامت نحو نموذج `Gemini` العالمي لضمان إنجاز التقارير وعدم توقف العمل أياً كانت الظروف.

## ⚙️ الموجه التفاعلي (The Interactive Selector)

- **انتقاء المكاتب المستهدفة**: واجهة سطر أوامر تفاعلية تسمح باختيار مكاتب محددة لمعالجتها باستخدام معرّفات رقمية بسيطة (مثلاً: `1 3`) أو إدخال `all` لمعالجة كافة الداتا دفعة واحدة.
- **تخصيص شهر التقييم**: تجاوز آلي لتاريخ التقرير عبر إدخال رقم الشهر (1-12). يترجم النظام الإدخال تلقائياً لأشهر المشرق العربي (مثال: `4 ➔ نيسان`)، ويقوم بمزامنة الاسم ضمن أسماء ملفات (`.docx`)، صفحات الغلاف، والمجلدات المنشأة على Google Drive لضمان الامتثال التام للمعايير الإدارية (القاعدة 8 من سجل المهارات).

## ⚠️ قاعدة الجراح (The Surgeon Rule)

للحفاظ على استقرار هذه الدقة المعمارية، تُطبّق **"قاعدة الجراح"** بصرامة:
*التعديلات البرمجية يجب أن تتم كتدخل جراحي دقيق وموضعي فقط في القسم المستهدف. يُمنع منعاً باتاً تغيير منطق المعالجة المتوازي الثابت (Orchestrator)، مسار الطوارئ (Failover)، أو المساس ببنية قواميس استخراج البيانات وتوجيهات النظام ما لم يتم طلب ذلك صراحةً.*

## 🔧 دليل التثبيت وإعداد البيئة (Setup Guide)

تعتمد المنظومة على بيئة Miniconda لإدارة الاعتماديات البرمجية بشكل نقي ومعزول:

**1. استنساخ المستودع:**
```bash
git clone <repository_url>
cd <repository_name>
```

**2. تهيئة بيئة Miniconda الخاصة بـ الذكاء الاصطناعي وتفعيلها:**
```bash
conda env create -f environment.yml
conda activate ai_env
```

**3. إعداد بيانات الاعتماد والمفاتيح السرية (Credentials):**
تأكد من وضع ملفات الاعتماد الحساسة في المجلد الجذري للمشروع (تتجاهلها المنظومة تلقائياً ولن تُرفع للإنترنت بفضل إعدادات الأمان الصارمة المُجهزة مسبقاً في `.gitignore`):
- إنشاء ملف `.env` وإضافة مفاتيح الذكاء الاصطناعي بداخله (`GROQ_API_KEYS=key1,key2` و `GEMINI_API_KEY=key`).
- توفير ملف `credentials.json` (ملف مصادقة Google Cloud OAuth2 السري).
- سيتم توليد ملف الوصول `token.json` (أو `token.pickle`) آلياً عند تشغيل المنظومة والمصادقة على الحساب للمرة الأولى عبر المتصفح.

**4. تشغيل منصة العمل:**
```bash
python main.py
```
اتبع التعليمات التفاعلية الواضحة على واجهة أوامر التشغيل (Terminal) لاختيار المكاتب وتحديد شهر التقييم واسمح لمحرك الذكاء الاصطناعي بتولي المهمة! 🤖
